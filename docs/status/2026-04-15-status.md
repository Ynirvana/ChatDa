# ChatDa — 현재 상황 (2026-04-15)

> 한국에 거주/방문하는 외국인을 위한 소셜 미팅 플랫폼.
> "Meetups for foreigners in Korea"

---

## 핵심 플로우

1. Google OAuth 로그인
2. 온보딩 (이름 · 국적 · 신분 · 소개 · 소셜링크 · 프로필 사진)
3. 미팅 둘러보기 → 참가 신청
4. 호스트가 신청자 프로필 보고 승인/거절
5. 미팅 종료 후 사진/후기 공유 (Memories)
6. 자유 게시판 (Feed)에서 댓글/대댓글로 소통

---

## 구현 완료 기능

### 인증 & 프로필
- Google OAuth (NextAuth v5) — 첫 로그인 시 DB user 자동 생성
- 온보딩 폼: 이름, 국적, 신분(tourist/student/expat 등), 소개, 소셜링크, 프로필 사진
- 프로필 사진 — base64로 DB 저장 (`users.profile_image`), Google 기본 사진 fallback + "Reset to Google photo"
- 프로필 수정 — 온보딩 폼 재사용 (initialData prop)
- **세션 콜백**: 매 세션마다 DB의 `profile_image`를 읽어와 `session.user.image` 덮어쓰기 → Nav 아바타에 최신 프사 반영

### 미팅 (Events)
- 모든 로그인 유저가 미팅 호스팅 가능 (HOST_EMAIL 제한 없음)
- **CreateEventForm** 필드:
  - 커버 사진 (base64 업로드)
  - 제목, 날짜, **시작/종료 시간** (`<input type="time">`)
  - 장소(venue name) + Google Maps URL + Naver Maps URL
  - **"How to find us"** — 지하철 출구/랜드마크 안내 (외국인 배려)
  - 지역(area), 정원, 입장료
  - **결제 방식** — dutch / split / cover / included
  - **요금 부연 설명** ("includes 2 shots + beer")
  - 설명
  - **참석 요건** — profile_photo / instagram / linkedin / facebook / x / tiktok 체크박스
- **이벤트 수정** — `/host/events/[id]/edit` (호스트만, PATCH 엔드포인트)
- 호스트 본인은 이벤트 상세 페이지 타이틀 옆 "Edit" 버튼

### 미팅 목록 & 상세
- `/meetups` — **Upcoming / Past 탭** 분리 (백엔드 `?past=true` 쿼리)
- 우측 하단 **FAB +** 버튼 (로그인 시) → 호스트 페이지로
- 이벤트 상세:
  - 커버 이미지 hero (그라데이션 fade)
  - 날짜/시간 + **"Add to Calendar" 버튼** (.ics 다운로드, KST→UTC 변환)
  - 장소 + 지도 링크 + "How to find us" 박스
  - 정원/요금/결제방식
  - 참석 요건 뱃지
  - **호스트 카드** (프사, bio, 소셜 링크)
  - **Pending Requests** (호스트 본인 + 다가오는 이벤트일 때만 노출)
    - 신청자 프로필 + 메시지 → Approve/Decline 바로 처리
  - 참석자 카드 리스트
  - 하단 sticky 바: RSVP 버튼 + Share 버튼 (Web Share API + 클립보드 fallback)
  - 지난 이벤트는 RSVP 자리에 "This meetup has ended" 표시

### Memories (이벤트 후기)
- 이벤트 날짜 지난 후에만 노출
- **호스트 + approved 참석자만** 작성 가능 (누구나 읽기)
- 사진 최대 10장 + 글 1000자
- PhotoGrid (1장: 와이드, 2장: 반반, 3장+: 3열)
- 라이트박스 (Esc/←/→ 키보드)
- 본인 글 삭제 가능

### Feed (자유 게시판)
- 글 작성 (1000자), 좋아요 (optimistic update)
- 댓글 + 대댓글 — 대댓글의 대댓글은 같은 top-level로 평탄화 (Twitter/Insta 방식)
- 본인 글/댓글 삭제

### 프로필 페이지
- **My Meetups** — 호스팅 + RSVP 모두 통합 리스트, 날짜순 정렬
  - 호스팅: `Hosting` 주황색 배지
  - RSVP: `Pending`/`Approved`/`Rejected` 배지

### Nav
- 텍스트 로고: `ChatDa` (그라데이션)
- Meetups, Feed 탭
- 우측 아바타 → 프로필 / 로그인 버튼

## 데이터 모델

### `users`
id, name, email, googleId, nationality, bio, profileImage, status, onboardingComplete, createdAt

### `socialLinks`
id, userId, platform (linkedin/instagram/x/tiktok/snapchat/whatsapp/kakao/facebook), url

### `events`
id, title, date, time, endTime, location, area, capacity, fee, description, coverImage, googleMapUrl, naverMapUrl, **directions**, requirements (JSON), paymentMethod, feeNote, hostId, createdAt

### `rsvps`
id, eventId, userId, status (pending/approved/rejected/cancelled), message, createdAt

### `posts` / `postLikes` / `postComments`
- postComments: id, postId, userId, **parentId** (nullable, 자기참조), content

### `eventMemories`
id, eventId, userId, content, **photos** (JSON array of base64), createdAt

## 백엔드 API (FastAPI · 포트 8001)

### 라우터
- `events` — GET /events (?past=true), GET /events/{id} (host + attendees + directions 포함)
- `host` — POST /events, PATCH /events/{id}, GET /pending (?event_id=), PATCH /rsvp, GET /events
- `rsvp` — POST /rsvp, GET /rsvp/status
- `users` — POST /onboarding, GET /me (rsvps + **hosted_events** 포함)
- `feed` — posts + likes + comments
- `memories` — GET /events/{id}/memories, POST (host/approved만), DELETE

### 인증
- `lib/proxy.ts` → `proxyToBackend(req, path, method, body)` → 매 요청마다 fresh JWT (HS256, 5분 TTL) 발급
- 백엔드는 `get_current_user_id`로 검증
- 공개 GET (feed, memories) + 인증 필요 mutation 구분

## 기술 스택

- **Frontend**: Next.js 15 (App Router, server components 기본), React, Inline styles
- **Auth**: NextAuth v5 (Google OAuth)
- **DB**: PostgreSQL (Docker, 포트 5434)
- **ORM**: Drizzle (마이그레이션) + SQLAlchemy async (백엔드 쿼리)
- **Backend**: FastAPI + asyncpg
- **이미지 저장**: base64 → DB Text 컬럼 (MVP — 추후 S3/Cloudinary 이전 예정)
