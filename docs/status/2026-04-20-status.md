## 구현 완료 기능

### 인증 & 게이트

- **Google OAuth (NextAuth v5)** — 비밀번호 없음, 첫 로그인 시 DB 자동 생성
- **Invite-only 게이트** — `signIn` 콜백에서 초대 토큰 쿠키 검증, admin 이메일은 자동 통과
- **초대 링크 흐름** — `/invite/[token]` → 쿠키 저장 → 로그인 → 자동 claim
  - 48시간 TTL, `invite_number` (serial), `note` 필드, 100명 cap (`INVITE_CAP`)
  - 상태: valid / expired / claimed / not_found 분기 UI
- `/join` — 초대 없는 방문자용 안내 (Threads DM 유도)
- Admin: `ADMIN_EMAILS` env 화이트리스트 (DB role 컬럼 없음)

### Onboarding Step 1

파일: `app/onboarding/page.tsx`, `components/OnboardingForm.tsx`, `app/api/onboarding/complete/route.ts`

| 필드 | 조건 |
|------|------|
| Display name | 필수 |
| Profile photo | 필수 (최대 5장 멀티 업로드, base64) |
| Gender | 필수 (Male / Female / Other) |
| Nationality | 필수 (196개 combobox, 유명도 순) |
| Status | 필수 (Student / Visitor / Resident; Korean 이면 Local 자동) |
| Location in Korea | 필수 (14개 광역) |
| Location District | Seoul 선택 시 sub-picker (25개 구, 선택 옵션) |
| School | Student일 때 필수 (22개 프리셋 + free text) |
| Age | 선택 (18~99 정수) |
| What brings you here | 선택 (10개 프리셋, 최대 3개) |

- 재방문 시 `?edit=1` 파라미터로 수정 모드

| 컴포넌트 | 기능 |

### People 탐색

파일: `app/people/page.tsx`, `components/PeopleClient.tsx`, `components/PersonCard.tsx`

**필터 (커스텀 FilterSelect, 네이티브 select 금지):**

| 필터 | 상태 |
|------|------|
| Location (14개 광역) | ✅ |
| Nationality (196개 combobox) | ✅ |
| Status | ✅ |
| What they offer (can_do 태그) | ✅ |
| Language | ✅ |
| What brings them (10개 프리셋) | ✅ |
| Has social (플랫폼별) | ✅ |
| School / District / Gender / Age | ❌ 백로그 |

**PersonCard:**
- Status별 커버 그라데이션
- 원형 아바타 + 국기 이모지
- Age·Gender 메타 (show_personal_info 반영)
- 체류 라인 (stay-duration.ts 로직)
- 🎓 School 뱃지 (Student만)
- Mutual connections count
- Outline Connect 버튼

**People 상세** (`app/people/[id]/page.tsx`):
- 이름/상태 + Age·Gender + 🎓 School 배지
- 체류 블록 + "What brings them here"
- Languages / Interests / Tags
- Social 링크 (1촌 연결 후에만 공개)
- 호스팅 이벤트 목록

**Peek 모드**: 미인증/미온보딩 유저도 목록 조회 가능 (상세 접근은 로그인 유도)

### 1촌 시스템 (Connections)

- 요청(pending) → 수락/거절 흐름
- `ConnectionRequests.tsx` — 받은 요청 목록
- PersonCard / 상세 페이지에 연결 상태 반영
- 1촌 mutual count 표시

### 소셜 링크

- 플랫폼: facebook, instagram, threads, x, linkedin, tiktok
- 1촌 연결 후에만 프로필에서 공개
- DB: `social_links` 테이블 (userId FK, platform enum, url)
- Profile completeness 항목에 포함

### Admin 페이지

파일: `app/admin/page.tsx`, `components/admin/AdminClient.tsx`

- 유저 목록 + 삭제 (self-protect: 본인 및 다른 admin 삭제 금지)
- 이벤트 목록 + 삭제
- 최신 posts/comments/memories (moderation)
- 초대장 관리 (생성/목록/삭제)
- 차단 이메일 관리

## DB 스키마 주요 테이블

### `users`

| 컬럼 | 타입 | 비고 |
|------|------|------|
| id, name, email, googleId | — | PK, unique |
| nationality, location, locationDistrict | text | Step 1 |
| status | enum | exchange_student / visitor / expat / local |
| school | text | Student 필수 |
| gender | enum | male / female / other |
| age | integer | 18~99 |
| bio | text | 100자 |
| languages | jsonb | [{lang, level}] |
| interests | text[] | 최대 10개 |
| profileImage | text | legacy (single) |
| profileImages | text[] | 최대 5장 |
| looking_for | text[] | 최대 3개 |
| looking_for_custom | text | 30자 |
| countryOfResidence | text | default 'KR' |
| show_personal_info | boolean | default true |
| onboarding_complete | boolean | — |
