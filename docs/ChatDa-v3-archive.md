## 1. 서비스 한 줄 요약

한국 거주 외국인·여행객·한국인을 위한 소셜 네트워킹 플랫폼. 외국인을 로컬 맛집에 직접 데려가는 모임에서 시작해, 일자리·생활정보·로컬 가이드까지 커버하는 "한국 사는 외국인의 홈". 오픈채팅방이 하고 있는 역할을 더 안전하고 체계적으로 대체한다.

최종 비전은 "한국에 관한 외국인 대상 정보는 전부 chatda에서". 거주자가 콘텐츠를 만들고, 여행객이 소비하는 구조. **한국인은 글로벌 네트워킹 파트너이자 로컬 가이드로 참여한다.**

---

## 2. 해결하려는 문제

한국에 외국인 278만 명이 살고 있고, 연간 관광객까지 합치면 약 1,900만 명인데, 이 사람들을 위한 통합 플랫폼이 없다.

지금 현실은 일자리는 카톡 오픈채팅방, 비자 정보는 레딧, 모임은 페이스북 그룹, 생활 질문은 또 다른 카톡방, 맛집은 구글맵(한국에서 제대로 안 됨). 전부 중구난방으로 흩어져 있어서 외국인들이 5~6개 채널을 떠돌아야 하는 상황.

모임 하나 찾으려고 해도 Meetup.com 들어가보고, 카톡 오픈채팅방 뒤지고, 페이스북 그룹 검색하고, 인스타에서 해시태그 찾고, Eventbrite도 확인하고. 이 파편화 자체가 가장 큰 문제.

정보 비대칭이 양방향으로 존재한다. 외국인은 한국에 뭐가 있는지 모르고(정보가 한국어이거나 흩어져 있어서 접근 불가), 한국인/사업자는 외국인 고객에게 어떻게 닿는지 모른다(외국인 대상 광고 채널이 사실상 없음). chatda는 이 양쪽 비대칭을 중간에서 연결한다.

**한국인 입장의 문제도 있다.** 글로벌 네트워킹을 하고 싶은 한국인(창업가, 프리랜서, 영어 사용 직장인)은 외국인을 만날 채널이 언어교환 모임 정도밖에 없다. 언어교환은 핑계일 뿐, 진짜 원하는 건 글로벌 인맥 확장인데, 그걸 직접적으로 제공하는 플랫폼이 없다.

실제 사례로 Sarah(이탈리아, 압구정 거주)는 한국에 혼자 와서 한 달째 거의 혼자. 외국인끼리도 이미 그룹이 형성되면 들어가기 어렵고, 홍대에서 성추행 위험을 경험해서 그쪽은 가지도 않음. 결국 오픈채팅방에서 낯선 사람이랑 대화하다가 카톡 교환하는 게 유일한 방법. 이런 사람이 수십만 명.

## 3. 시장 크기

TAM은 체류 외국인 278만 + 연간 방한 관광객 약 1,600만 = 약 1,900만 명.

SAM은 일자리 수요가 있는 취업자격 59만 + 유학생 31만 = 약 90만 명. 유학생 90%가 월 200만 원 미만이라 단기 알바 수요 높음. 여기에 여행객 중 로컬 정보를 찾는 층까지.

SOM은 서울/수도권 영어 사용 가능 외국인 약 15~20만 명. 단기 일자리 수요가 있는 유학생·프리랜서 위주로 초기 타겟 5~8만 명.

확장 여지는 명확. 영어로 시작하되, 중국어·베트남어만 추가하면 SAM의 47%(중국 35% + 베트남 12%)를 바로 흡수 가능.

주요 수치로 체류 외국인의 90.4%가 한국 계속 체류 희망. 주요 국적은 중국 35%, 베트남 12%, 미국 7%, 태국 6%, 우즈베키스탄 4%. 유학생은 전년 대비 17% 증가 추세.

## 4. 경쟁사 조사

### 4-1. 일자리 경쟁자

가장 위협적인 일자리 경쟁자는 K-HIRE(알바천국). 2025년 12월 출시 후 두 달 만에 MAU 12만 명 돌파, 공고 2만 7천 건. 채용 + 커뮤니티 통합 플랫폼. 알바천국이라는 거대 모회사의 공고 물량과 자본이 강점.

기타 잡보드로 KOWORK(비자~취업 원스톱, "No.1" 표방), K-Work(정부 운영, 유학생 특화), WithKorea24(비자 상담 + 일자리 + 커뮤니티 통합), KLiK(200만 외국인+유학생 타겟), Buddies(유학생 특화, 이력서·자소서·면접 가이드), Jobploy(외국인 채용 매칭), Enkorwithus(외국인 방문자 대상 커뮤니티 + 숙소).

### 4-5. 핵심 인사이트

사람들이 YNA 같은 모임에 가는 진짜 이유는 언어 교환이 아니라 새로운 사람을 만나고 싶어서. 언어 교환은 핑계. 애프터파티 90% 참석이 증거. 사람들한테 필요한 건 "만남의 명분"이다. chatda는 그 명분으로 "음식"을 쓴다.

Eventbrite 서울 검색 결과에서도 캐주얼 음식 모임은 단 하나도 없음(2026년 4월 기준). 이 빈자리를 chatda가 잡는다.

우리 포지셔닝은 K-HIRE와 정면 경쟁 아님. 오픈채팅방 대체. 아무도 안 하고 있는 영역인 "외국인을 로컬 맛집에 데려가면서 사람을 만나게 해주는 것"을 잡는다.

## 5. 핵심 컨셉 — 소셜 노드

유저가 새 프로필을 처음부터 만드는 게 아니라, 기존 소셜미디어(LinkedIn, Instagram, X, TikTok, 카카오톡)를 연결하고 비자 유형·국적·거주지 같은 실용 정보만 추가하는 구조. 한 번 만들면 구직·모임·커뮤니티에서 전부 재활용. 이력서 대신 소셜미디어 링크로 "이 사람 어떤 사람인지" 바로 파악 가능.

모임에서 "누가 오는지" 프로필을 미리 볼 수 있는 게 오픈채팅방 대비 핵심 차별점. **이것은 안전 기능이기도 하다 — 프로필이 공개되면 이상한 행동을 하기 어렵다.**

## 6. 인증 시스템 — 3트랙 + 소셜미디어 필수

chatda의 멤버 퀄리티와 안전을 보장하는 핵심 구조. 오픈채팅방과의 결정적 차별점.

### 6-1. 공통 필수 요건 (전원)

**소셜미디어 최소 1개 연동 필수.** LinkedIn, Instagram, X(Twitter), TikTok 중 택 1 이상.

자동 필터 최소 기준:

- 계정 생성 6개월 이상
- 게시물 5개 이상
- 팔로워 10명 이상

이 기준을 통과하지 못하면 가입 보류 → "소셜미디어를 업데이트 후 다시 시도하세요" 안내. 이 기준은 가짜 계정·스팸·이상한 목적의 가입을 걸러내기 위한 것.

### 6-2. 트랙 = 한국에서의 상태 (국적 아님)

**트랙은 "인증 방식"이지 "국적"이 아니다.** 한국인도 Student 트랙으로 가입할 수 있고, 외국인도 Local 트랙으로 가입할 수 있다. 국적은 프로필에서 따로 입력받는 별도 필드.

- **Student** = "나는 대학 이메일로 인증할 수 있다" (한국인이든 외국인이든)
- **Local** = "나는 한국 폰번호로 인증할 수 있다" (한국인이든 장기 거주 외국인이든)
- **Traveler** = "둘 다 없다" (단기 여행객, 디지털노마드 등)

| 트랙 | 의미 | 추가 인증 방식 | 승인 방식 |
| --- | --- | --- | --- |
| **Student** | 한국에서 공부 중 | 대학 이메일 (.edu / .ac.kr) | 자동 (이메일 인증 코드) |
| **Traveler** | 단기 방문 | 소셜미디어가 본인 인증 겸함 | 자동 (소셜미디어 기준 통과 시) |
| **Local** | 한국에 거주 중 | 휴대폰 번호 인증 (010) | 자동 (SMS 인증 코드) |

프로필 표시 예시: "Student @ 고려대 · Korean", "Local · French", "Traveler · German"

### 6-3. 향후 인증 강화 — ARC 인증 (메모)

### 6-4. 설계 원칙

입구는 3개(Student / Traveler / Local), 들어오면 같은 파티장. 모든 이벤트에 세 트랙 유저가 함께 참여. 프로필에 트랙 태그 + 국적이 표시되어 서로 배경을 알 수 있음.

Traveler 트랙은 의도적으로 진입장벽이 가장 낮음. 여행객은 체류 기간이 짧아서 인증이 복잡하면 이탈. 대신 소셜미디어 자동 필터가 퀄리티를 유지.

이벤트에서 외국인/한국인 비율을 관리하고 싶으면 트랙이 아니라 **국적 필드 기준으로 쿼터**를 건다. (예: 이벤트당 외국인 70% + 한국인 30%)

**MVP에서는 Google OAuth로 계정 생성 후, 소셜미디어 프로필 URL은 직접 입력 + 수동 검증으로 시작. 소셜미디어 OAuth 연동은 유저 풀이 쌓인 후 자동화.**

## 7. chatda.life MVP — 제품 기획

### 7-1. MVP 범위

**포함:**

- 랜딩 페이지 (서비스 소개 + 가입 유도)
- Google OAuth 로그인 / 이메일 폴백
- 회원가입 / 인증 플로우 (3트랙)
- 프로필 (이름, 국적, 트랙 태그, 소셜미디어 링크, 한 줄 소개, 관심 분야)
- 이벤트 리스트 (다가오는 디너 모임 목록)
- 이벤트 상세 (장소, 시간, 참가비, 참가자 프로필 목록)
- 참석 신청 (RSVP 버튼)

**제외 (Phase 2 이후):**

- 채팅 / DM
- 멤버 매칭 / 추천
- 커뮤니티 피드 / Q&A
- 결제 시스템 (참가비는 토스/카카오페이 외부 정산)
- 자영업자 비즈니스 페이지
- 알바 / 채용 포스팅
- 알림 / 푸시

## 8. PMF 진입점 — 직접 모임 주최 (맛집 + 펍)

chatda의 첫 번째 역할은 외국인을 로컬 맛집과 펍에 직접 데려가는 모임을 여는 것.

### 8-1. 두 가지 포맷

**포맷 A: chatda eats — 삼겹살 등 로컬 맛집 탐방.** chatda의 시그니처. "외국인을 진짜 한국 음식에 데려간다"는 포지셔닝. 진입장벽이 낮음("삼겹살 먹으러 가자"는 "영어 연습하자"보다 100배 부담 없음). 콘텐츠가 자동으로 나옴(음식 비주얼 + 외국인 리액션 = 인스타 콘텐츠).

**포맷 B: chatda nights — 펀마이마이 등 펍/게임.** 맥주 + 게임이 한 곳에서 되는 장소 활용. 삼겹살보다 캐주얼하고 시간 제약이 적음. 다른 분위기로 리피터가 안 질림.

두 포맷을 번갈아 돌리면 매주 다른 분위기 + 콘텐츠 다양성 확보.

### 8-2. 한국인(Local)의 역할

**Local 유저는 "로컬 가이드" 역할을 겸한다.** 식당에서 주문 도움, 문화 설명, 분위기 연결. 이건 chatda만의 차별점 — YNA/Jake/S.H.Y. 모임에는 한국인이 공식적으로 참여하는 구조가 없다. 한국인 입장에서도 "글로벌 네트워킹 + 맛집"이라는 명분이 있으니 참여 동기가 생긴다.

### 8-3. 금전 처리

참가비: 초기 ₩1,000(노쇼 방지). 매회 10명 이상 꾸준히 오면 ₩3,000으로 인상. 참가비는 사전에 토스/카카오페이로 송금.

삼겹살(포맷 A) 음식값: 주최자가 일괄 결제 → N빵 정산. 삼겹살은 고기를 같이 구워 먹으므로 개별 계산 불가. 현장에서 영수증 찍고 "총 ₩OO, N명이니 인당 ₩OO" 공유.

펍(포맷 B) 음식값: 각자 계산.

외국인 결제 수단: 사전 ₩1,000 송금을 통과한 사람 = 토스/카카오페이 가능 확인 완료. 안 되는 사람은 현금 허용.

### 8-4. 참가 신청 흐름

1. chatda.life에서 이벤트 확인 + RSVP (Eventbrite에도 동시 등록해 검색 유입 확보)
2. RSVP 후 안내: "참가 확정하려면 24시간 내 토스/카카오페이로 ₩1,000 보내주세요" + 송금 링크
3. 송금 확인 → WhatsApp 그룹 초대 + 장소/시간 상세 안내
4. 미송금 → 자동 탈락 (노쇼 방지)

### 8-5. 주최자 직접 참석 원칙

최소 처음 3~4회는 직접 참석 필수. 이유: (1) 분위기 잡기 (2) 외국인 주문/소통 도움 (3) 사진/영상 촬영 → 콘텐츠 (4) "외국인 친구 많은 한국인" 브랜드 = 그 한국인이 있어야 신뢰. 리피터가 생기면 단골 참가자에게 호스트 위임 가능.

## 10. 제품 로드맵

### Phase 1 — MVP (0~2주)

chatda.life에 가입/인증(3트랙) + 프로필 + 이벤트 RSVP 세팅. Eventbrite에 무료 티켓으로 이벤트 동시 등록. 인스타/스레드에 공감형 콘텐츠 매일 1개. 오픈채팅방, 페북 그룹, 와츠앱에서 첫 삼겹살 모임 참가자 모집.

**목표:** 첫 모임 3~5명 소규모 테스트.

### Phase 2 — 반복 검증 (2~4주)

삼겹살 + 펍을 번갈아 주 1회 운영. 모임 사진/영상이 곧 인스타 콘텐츠. 참가자가 자발적으로 친구를 데려오는지 확인.

**목표:** 매회 10명 이상 안정화 → 참가비 ₩3,000 도입.

### Phase 3 — 생태계 확장 (1~2개월)

커뮤니티 Q&A (지식인 스타일) 추가. 멤버 디렉토리 (서로 검색/연결). 다른 모임(YNA 등) 정보 큐레이션 병행. 펍/식당 사장님과 B2B 딜 시작.

**목표:** 활성 유저 500명, B2B 딜 1곳 이상.

### Phase 4 — 테마 모임 + 플랫폼화 (2~3개월)

참가자 풀에서 수요 확인된 세그먼트별 모임 신설 (국적별, 창업가, 취미별). 리피터에게 호스트 위임. chatda.life를 누구나 모임을 올리고 참가할 수 있는 구조로 전환.

**원칙: 수요를 확인한 다음 모임을 만든다. 모임을 먼저 만들고 수요를 찾지 않는다.**

### Phase 5 — 수익화 확장 (3~6개월)

자영업자 비즈니스 페이지, 알바/채용 포스팅 추가. 프리미엄 체험 모임 연결.

### Phase 6 — 풀 플랫폼 (6개월~)

일자리 매칭, 여행객 확장, 생활 서비스 연결, 로컬 광고.

## 13. 콘텐츠 전략

### 채널 정체성

"외국인 친구 많은 한국인이 운영하는 한국 생활 치트키 채널". 톤은 가볍고 캐주얼, 영어 기반에 한국어가 자연스럽게 섞이는 느낌. lol, lmao, ngl 같은 표현 쓰는 게 오히려 자연스러움. 관공서 느낌 절대 아님.

### 네 가지 축

**축 1: 공감/유머 (50%)** — 외국인이 "ㅋㅋ 맞아" 하면서 공유하는 글. 팔로워를 키우는 엔진. 초기 2주는 80%로.

**축 2: 로컬 꿀팁 (25%)** — 저장하고 공유하는 실용 정보. 캐러셀이 저장률 높음.

**축 3: 연결/모임 떡밥 (15%)** — "만남의 기대감"을 심는 축. 팔로워 500명 넘긴 후 본격.

**축 4: 모임 후기 (10%)** — chatda eats & nights 후기. 사진/영상에서 자연스럽게 나오는 시그니처 콘텐츠.

### 규칙

매일 1개, 최소 주 4~5개. 모든 글 끝에 댓글 유도 한 줄 필수. 스레드는 3~5줄이 최적. "팔로우 해주세요" 금지. 사람이 쓴 느낌으로.

## 14. 리스크

**닭과 달걀 문제.** 직접 모임을 열어서 유저 풀을 먼저 만들고, 모임에서 나오는 사진/영상으로 콘텐츠를 채운다.

**불법취업·비자 문제.** 일자리 매칭 단계에서 법적 리스크. 면책 고지 + 신고 시스템으로 초기 방어.

**K-HIRE와의 경쟁.** 일자리에서는 공고 물량으로 못 이김. 밋업 → 커뮤니티 → 일자리 순서로 다른 게임.

**인증 시스템 우회.** 소셜미디어 자동 필터 기준을 통과하는 가짜 계정. MVP에서는 수동 모니터링으로 보완, 유저 풀 확대 후 신고 기능 + 자동화 강화.

**모임 확장 리스크.** 수요 확인 전에 테마 모임을 벌리면 에너지 분산. 원칙: 참가자 풀에서 세그먼트가 보일 때만 신설.

**한국인 참여 밸런스.** 한국인이 너무 많으면 "한국인 모임"이 되고, 너무 적으면 "로컬 가이드" 역할이 안 됨. 초기 목표: 이벤트당 외국인 70% + 한국인 30% 비율 유지. 트랙이 아닌 국적 필드 기준으로 쿼터 관리.

## 15. 확장 방향

**테마 모임:** 참가자 풀에서 수요 확인된 세그먼트별 확장. 창업가 미팅, 국적별 모임, 취미별 모임. 리피터에게 호스트를 맡겨 chatda는 플랫폼 역할로 전환.

**자영업자 페이지:** 사장님이 본인 매장을 소개하고 외국인 대상 프로모션 게시.

**일자리:** 단기 알바 포스팅 + 매칭.

**커뮤니티 Q&A:** 네이버 지식인 스타일. 비자, 생활, 맛집 등 외국인이 묻고 답하는 공간.

**여행객 확장:** 거주자가 만든 로컬 콘텐츠를 여행객에게 연결. 로컬 광고, 생활 서비스 연결로 수익 확장.
