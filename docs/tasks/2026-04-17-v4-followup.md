## 2. "한국에 있는 기간" 표시 강화

### 현재 상태
- `users.stayArrived` / `stayDeparted` (date) — 이미 스키마에 있고 `StayDatesEditor.tsx`에서 프로필 페이지 입력 가능.
- status별 분기도 구현됨 (Expat=Since when / Visitor=arrived+departed / …).
- **그런데 People 카드/상세에서 "얼마나 있었는지 / 얼마나 더 있을 건지"가 표시되지 않음** → 탐색 시 중요 맥락 누락.

### 방향 (확정)
**A (표시 강화)**, 단 **Expat은 month precision 입력**으로 조정.
- Expat은 정확한 도착일 기억 못 하거나 공개 부담 큼 → `<input type="month">`로 받고 서버 저장 시 `day=01`. 수치 계산(years/months)은 그대로 동작.
- Visitor / Visiting soon / Visited before는 날짜 입력 유지 (Day 7 of 14, Arriving in 12 days 같은 카운트다운 의미 있음).

### 설계 (옵션 A)
- `lib/stay-duration.ts` **신규** — 순수 함수:
  - `formatStay(status, arrived, departed, now)` → `{ primary: string, secondary?: string }`
  - Expat + arrived: `"2y 3mo in Korea"` (primary), `"Since Jan 2024"` (secondary)
  - Visitor + arrived + departed: `"14 days (Apr 5 – Apr 19)"`; 현재 진행 중이면 `"Day 7 of 14"`
  - Visiting_soon + arrived 미래: `"Arriving in 12 days"` 강조 오렌지
  - Visited_before: `"Visited 3mo ago (7 days)"`
- 노출 위치:
  - `components/PersonCard.tsx` — status 배지 옆 작은 메타 라인
  - `app/people/[id]/page.tsx` — 상단 스탯 영역
  - `app/profile/page.tsx` — 본인 프로필에도 미리보기
- `components/StayDatesEditor.tsx` — status === 'expat' 분기 추가 → month picker. 나머지 상태는 date picker 유지.
- **People 필터 추가 (선택)**: "Staying this month" / "Staying 1+ year" 같은 유효기간 기반 필터. 이건 P2로 뒤로.

### DB
변경 없음. 기존 컬럼 활용.

### 리스크
- 날짜 미입력 유저 다수 → fallback "duration unknown" 처리. 빈 값을 드러내면 오히려 완성도 유도 효과 있음 (ProfileCompleteness와 상호보완).

## 5. DM 기능 (별도 PR 권장)

### 목표
연결된 두 유저(connection accepted) 간 1:1 메시지 송수신. 그룹 DM 아님.

### 범위 결정
**v0 (이 계획 커버 범위 — 확정)**:
- 1:1 텍스트만, **body 최대 2000자**
- Connect accepted 관계만 DM 가능 (남발 방지)
- **폴링 계층화**:
  - 열린 대화창: 10s
  - 인박스 페이지: 30s
  - Nav ✉️ 뱃지 (전역): 60s
- 읽음 표시 (at 레벨, `conversation_reads.last_read_at`)
- Nav엔 ✉️ 뱃지만 신설. 🔔 알림 시스템은 별도 백로그라 DM과 독립적으로 나중에 옆자리에 붙임.

**v0에서 제외 (백로그)**:
- 이미지/파일 첨부
- 타이핑 인디케이터
- 그룹 DM
- 푸시 알림
- 검색
- 차단/신고 — admin-level은 P0, user-level block은 v1

### DB (신규 마이그레이션 `0018_*.sql`)

```sql
-- 대화 (두 유저 사이 1개만 존재)
CREATE TABLE conversations (
  id text PRIMARY KEY,
  user_a text NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  user_b text NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  last_message_at timestamp,
  created_at timestamp NOT NULL DEFAULT now(),
  -- user_a < user_b 정렬 강제 (unique 보장)
  CONSTRAINT user_order CHECK (user_a < user_b),
  UNIQUE (user_a, user_b)
);

-- 메시지
CREATE TABLE messages (
  id text PRIMARY KEY,
  conversation_id text NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
  sender_id text NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  body text NOT NULL,  -- 2000자 제한 (앱 레벨)
  created_at timestamp NOT NULL DEFAULT now()
);
CREATE INDEX ON messages (conversation_id, created_at DESC);

-- 읽음 포인터 (per-user per-conversation)
CREATE TABLE conversation_reads (
  conversation_id text NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
  user_id text NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  last_read_at timestamp NOT NULL DEFAULT now(),
  PRIMARY KEY (conversation_id, user_id)
);
```

### API (FastAPI `backend/routers/messages.py` **신규**)

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/messages/inbox` | 내 대화 목록 (상대방 프로필 + 마지막 메시지 + 미읽 수) |
| POST | `/api/messages/conversations` | 대화 시작/찾기 (body: `{recipientId}`) — connection check |
| GET | `/api/messages/conversations/{id}` | 특정 대화 메시지 목록 (paginated, before=cursor) |
| POST | `/api/messages/conversations/{id}` | 메시지 전송 (body: `{body}`) |
| POST | `/api/messages/conversations/{id}/read` | 읽음 표시 (현재 시각으로 `last_read_at` 업데이트) |
| GET | `/api/messages/unread-count` | Nav 뱃지용 |

**핵심 검증**:
- sender/recipient가 서로 `connections.status = 'accepted'`인지 확인. 아니면 403.
- 자기 자신 DM 금지.
- Body rate limit (추후 slowapi).

### 프론트

- `app/messages/page.tsx` **신규** — 인박스 (대화 목록). 좌측 리스트 + 우측 빈 상태. 30초 폴링.
- `app/messages/[conversationId]/page.tsx` **신규** — 대화창. 10초 폴링 `useEffect` + `setInterval`.
- `components/ui/Nav.tsx` — ✉️ 아이콘 + 미읽 뱃지 신설, `/api/messages/unread-count` 60초 폴링.
- `components/PersonCard.tsx` / `app/people/[id]/page.tsx` — Connect 완료 상태에선 "Message" 버튼도 노출 → POST `/api/messages/conversations` → 리다이렉트.
- **디자인 참조**: 기본 다크 테마 유지, 메시지 버블은 오렌지 그라디언트(내 거) / glass(상대). 타임스탬프는 묶음 단위.

### 리스크 / 결정 포인트

- **폴링 비용**: 활성 유저 100명이 각자 10초 폴링 → 초당 10 req. 지금 용량에선 문제 없음. 성장하면 SSE/WebSocket.
- **읽지 않은 수 계산 쿼리**: 대화별 `count(messages where created_at > last_read_at and sender_id != me)`. 인박스 10건 기준 N+1 주의 — 한 쿼리로 묶기.
- **대화 unique 제약**: `user_a < user_b` 조건 강제해서 (A,B)와 (B,A) 중복 방지. 대화 찾을 때도 두 id 정렬 후 쿼리.
- **Connection 끊어지면?**: 기존 대화 보존하되 새 메시지 전송 막기. 또는 "This user is no longer connected" 안내. **기본: 전송만 막고 기록 유지**.
- **DM이 스팸 온상 되면?**: connection 필수 정책이 1차 방어. 신고 기능은 v1.
- **GDPR/데이터 삭제**: 유저 탈퇴 시 ON DELETE CASCADE로 메시지도 삭제됨 — OK.
